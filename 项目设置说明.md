# 学业预警系统 - 项目设置说明

## 项目概述

学业预警与帮扶闭环平台是一个面向学生、教师、辅导员、管理员的综合性学业管理系统。系统支持学业预警触发、帮扶计划制定与跟踪、成绩申诉等功能。

## 技术架构

### 后端技术栈
- **框架**: Spring Boot 3.2.0
- **ORM**: MyBatis-Plus 3.5.4
java21
- **数据库**: MySQL 8.0
- **缓存**: Redis
- **认证**: JWT + Spring Security
- **构建工具**: Maven
- **其他**: Apache POI（Excel导出）、Java Mail（邮件通知）

### 前端技术栈
- **框架**: Vue 3.3.4
- **构建工具**: Vite 4.4.9
- **路由**: Vue Router 4.2.4
- **状态管理**: Pinia 2.1.4
- **HTTP客户端**: Axios 1.5.0
- **UI组件库**: Element Plus 2.4.0
- **图表**: ECharts 5.4.2
- **Node版本管理**: nvm

## 快速开始

### 1. 数据库初始化

#### 步骤1: 创建数据库
```bash
# 使用MySQL命令行工具
mysql -u root -p < 学业预警系统后端/src/main/resources/init_database.sql
```

#### 步骤2: 验证数据库连接
编辑 `学业预警系统后端/src/main/resources/application.yml`，配置以下内容：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/academic_warning_system?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=GMT%2B8
    username: root
    password: your_password  # 替换为实际密码
  redis:
    host: localhost
    port: 6379
```

### 2. 后端项目启动

#### 前提条件
- Java 17+
- Maven 3.6+
- MySQL 8.0+
- Redis 6.0+（可选，如果不使用缓存功能）

#### 启动步骤
```bash
# 进入后端项目目录
cd 学业预警系统后端

# 编译项目
mvn clean compile

# 启动应用
mvn spring-boot:run

# 或打包后运行
mvn clean package
java -jar target/academic-warning-system-1.0.0.jar
```

后端默认运行在 `http://localhost:8080`

### 3. 前端项目启动

#### 前提条件
- Node.js 16+ 或使用 nvm 管理版本
- npm 7+ 或 yarn

#### 启动步骤
```bash
# 进入前端项目目录
cd 学业预警系统前端

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 或使用yarn
yarn install
yarn dev
```

前端默认运行在 `http://localhost:5173`

## 安全与合规要点

1. **密码加密**: 统一采用 SHA-256 + 盐值加密，禁止明文存储
2. **外键约束**: 删除学院/专业前，清理关联引用，避免孤儿数据
3. **修改留痕**: 成绩修改写入 `score_logs`，保留原因和修改人
4. **隐私控制**: `student_profile.privacy_level` 控制成绩可见范围
5. **认证**: JWT（短期）+ 刷新令牌
6. **防护**: CSRF、XSS、SQL注入防护

## 常见问题

### Q: 如何配置Redis（如果不使用缓存）？
A: 在 `application.yml` 中移除Redis配置，系统会自动降级到数据库查询。

### Q: 学号解析失败如何处理？
A: 系统会记录在 `audit_logs` 中，管理员可在"专业编码管理"页面配置缺失的专业编码。

### Q: 如何导出成绩或帮扶计划？
A: 相关页面提供"导出"按钮，使用Apache POI生成Excel文件。

## 部署建议

### 开发环境
- 本地运行（如上述快速开始）
- 使用内存数据库H2或SQLite（可选）

### 生产环境
- 前端：使用Nginx反向代理，`npm run build` 生成静态资源
- 后端：使用容器化部署（Docker）或传统应用服务器
- 数据库：使用高可用MySQL集群
- 缓存：使用Redis集群
- 监控：集成Spring Boot Actuator + Prometheus
