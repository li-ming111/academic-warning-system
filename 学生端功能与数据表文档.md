# 学业预警系统 - 学生端功能与数据表文档

## 一、学生端功能概述

学生端提供完整的学业管理功能，包括智能看板、成绩分析、预警管理、帮扶计划、统计分析和个人设置六大核心模块。

---

## 二、核心功能模块

### 1. 智能看板 (Dashboard)
**功能描述**：提供学生学业数据的综合展示，包括GPA、预警统计、帮扶计划进度和通知提醒。

**涉及数据表**：
- `student_profile` - 学生基本信息
- `scores` - 成绩数据（计算GPA）
- `academic_warnings` - 预警信息
- `assistance_plans` - 帮扶计划
- `notifications` - 通知提醒

**关键字段映射**：
```sql
-- student_profile表
id, user_id, student_id, name, college_id, major_id, class_id, privacy_level

-- scores表
id, student_id, course_id, score_total, grade_point, semester

-- academic_warnings表
id, student_id, warning_level, description, status, appeal_status

-- assistance_plans表
id, student_id, warning_id, progress_percentage, status

-- notifications表
id, user_id, content, status, created_at
```

**API接口**：
- `GET /api/students/dashboard/{userId}` - 获取看板数据
- `GET /api/students/notifications/{userId}` - 获取通知列表

---

### 2. 成绩深度分析 (Scores)
**功能描述**：展示学生所有课程成绩，支持按学期筛选、成绩趋势分析和GPA计算。

**涉及数据表**：
- `scores` - 成绩主表
- `courses` - 课程信息
- `enrollments` - 选课记录

**关键字段**：
```sql
-- scores表
student_id, course_id, score_total, grade_point, semester, created_at

-- courses表
id, name, credits, type, score_template

-- enrollments表
id, student_id, course_id, semester, enrollment_date
```

**核心功能**：
- 成绩列表展示（课程名称、成绩、学分、绩点）
- 学期筛选
- GPA统计计算
- 成绩趋势图表

**API接口**：
- `GET /api/students/scores/{studentId}` - 获取成绩列表
- `GET /api/students/scores/{studentId}?semester={semester}` - 按学期筛选

---

### 3. 智能预警提醒 (Warnings)
**功能描述**：展示学业预警信息，支持预警详情查看、分级筛选和成绩申诉功能。

**涉及数据表**：
- `academic_warnings` - 预警记录
- `score_appeals` - 成绩申诉
- `courses` - 课程信息

**关键字段**：
```sql
-- academic_warnings表
id, student_id, warning_level (red/yellow/blue), description, status, appeal_status

-- score_appeals表
id, warning_id, reason, attachments, status (pending/approved/rejected)
```

**核心功能**：
- 预警统计（红色/黄色/蓝色预警数量）
- 预警列表展示（支持展开详情）
- 预警等级筛选
- 成绩申诉提交
- 申诉记录查看

**API接口**：
- `GET /api/students/warnings/{studentId}` - 获取预警列表
- `GET /api/students/appeals/{userId}` - 获取申诉列表
- `POST /api/students/appeals` - 提交申诉

---

### 4. 帮扶计划追踪 (Assistance)
**功能描述**：查看教师或辅导员为学生制定的帮扶计划，跟踪计划执行进度。

**涉及数据表**：
- `assistance_plans` - 帮扶计划
- `academic_warnings` - 关联的预警记录

**关键字段**：
```sql
-- assistance_plans表
id, student_id, warning_id, progress_percentage, status (initiated/in_progress/completed/cancelled)
```

**核心功能**：
- 帮扶计划列表
- 进度展示（进度条）
- 计划状态跟踪

**API接口**：
- `GET /api/students/assistance/{studentId}` - 获取帮扶计划列表

---

### 5. 个人信息管家 (Statistics)
**功能描述**：提供全面的学业数据统计分析，包括GPA趋势、成绩分布、预警统计等。

**涉及数据表**：
- `scores` - 成绩数据
- `academic_warnings` - 预警数据
- `courses` - 课程信息

**关键统计指标**：
```
1. GPA统计
   - 当前GPA
   - 平均GPA
   - 最高/最低GPA

2. 成绩分布
   - 优秀（≥85）门数
   - 良好（75-84）门数
   - 及格（60-74）门数
   - 不及格（<60）门数

3. 预警统计
   - 红色预警数量
   - 黄色预警数量
   - 绿色预警数量

4. 学期成绩趋势图
5. 课程成绩详情表
```

**API接口**：
- `GET /api/students/statistics/{userId}` - 获取统计数据

---

### 6. 个人设置 (Settings)
**功能描述**：管理个人信息、隐私设置、密码修改和安全日志查看。

**涉及数据表**：
- `student_profile` - 学生信息
- `users` - 用户账户
- `security_log` - 安全日志

**关键字段**：
```sql
-- student_profile表
name, student_id, college_id, major_id, class_id, privacy_level

-- users表
username, password, phone, email

-- security_log表
user_id, ip_address, login_time, is_successful, created_at
```

**核心功能**：
- 个人信息展示
- 隐私级别设置（0=公开，1=学院内可见，2=仅教师可见，3=仅本人与管理员）
- 密码修改
- 安全日志查看（登录记录）

**API接口**：
- `GET /api/students/settings/{userId}` - 获取用户设置
- `POST /api/students/settings/{userId}/change-password` - 修改密码
- `POST /api/students/settings/{userId}/privacy-level` - 更新隐私级别
- `GET /api/students/settings/{userId}/security-logs` - 获取安全日志

## 三、核心数据表详解

### 1. users（用户表）
存储系统所有用户的账户信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| username | varchar(20) | 用户名（唯一） |
| password | varchar(64) | 密码（加密） |
| role | tinyint | 角色（1=学生，2=教师，3=管理员，4=辅导员） |
| phone | varchar(11) | 手机号 |
| email | varchar(50) | 邮箱 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 2. student_profile（学生档案表）
存储学生的详细信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| user_id | int | 关联users表（外键，唯一） |
| name | varchar(100) | 姓名 |
| student_id | varchar(50) | 学号 |
| college_id | int | 学院ID |
| major_id | int | 专业ID |
| class_id | int | 班级ID |
| privacy_level | tinyint | 隐私级别（默认2） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

**关系**：`student_profile.user_id` → `users.id`

---

### 3. scores（成绩表）
存储学生所有课程成绩。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| student_id | int | 学生ID（关联student_profile.id） |
| course_id | int | 课程ID |
| score_total | decimal(5,2) | 总成绩 |
| grade_point | decimal(3,1) | 绩点 |
| semester | varchar(20) | 学期（如2025-12） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

**关系**：
- `scores.student_id` → `student_profile.id`
- `scores.course_id` → `courses.id`

---

### 4. courses（课程表）
存储所有课程信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | varchar(100) | 课程名称 |
| credits | decimal(3,1) | 学分 |
| type | varchar(10) | 类型（required=必修，elective=选修） |
| score_template | varchar(20) | 成绩模板（exam等） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 5. academic_warnings（学业预警表）
存储学生的预警记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| student_id | int | 学生ID |
| warning_level | varchar(10) | 预警等级（red/yellow/blue） |
| description | text | 预警描述 |
| status | varchar(20) | 状态（pending/confirmed/processed/closed） |
| appeal_status | varchar(20) | 申诉状态（none/pending/under_appeal/resolved/rejected） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 6. score_appeals（成绩申诉表）
存储学生的成绩申诉记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| warning_id | int | 预警ID |
| reason | text | 申诉原因 |
| attachments | varchar(255) | 附件路径 |
| status | varchar(20) | 状态（pending/approved/rejected） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 7. assistance_plans（帮扶计划表）
存储教师或辅导员为学生制定的帮扶计划。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| student_id | int | 学生ID |
| warning_id | int | 关联的预警ID |
| progress_percentage | tinyint | 进度百分比（0-100） |
| status | varchar(20) | 状态（initiated/in_progress/completed/cancelled） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

---

### 8. notifications（通知表）
存储系统通知消息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| user_id | int | 接收用户ID |
| content | text | 通知内容 |
| status | varchar(10) | 状态（unread/read） |
| created_at | datetime | 创建时间 |

---

### 9. security_log（安全日志表）
记录用户登录日志。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| user_id | int | 用户ID |
| ip_address | varchar(15) | 登录IP |
| login_time | datetime | 登录时间 |
| is_successful | tinyint | 是否成功（1=成功，0=失败） |
| created_at | timestamp | 创建时间 |

---

## 四、数据流转关系

```
用户登录（users表）
    ↓
获取学生档案（student_profile表，通过user_id关联）
    ↓
查询相关数据：
    ├─ 成绩数据（scores表，通过student_id关联）
    │   └─ 课程信息（courses表，通过course_id关联）
    ├─ 预警信息（academic_warnings表）
    │   └─ 申诉记录（score_appeals表，通过warning_id关联）
    ├─ 帮扶计划（assistance_plans表）
    ├─ 通知消息（notifications表，通过user_id关联）
    └─ 安全日志（security_log表，通过user_id关联）
```

## 五、技术实现

### 前端技术栈
- Vue 3 + Vite
- Element Plus（UI组件库）
- Axios（HTTP请求）
- Vue Router（路由管理）
- Pinia（状态管理）

### 后端技术栈
- Spring Boot 3
- MyBatis-Plus（ORM框架）
- MySQL 8.0
- Spring Security（安全框架）
- Redis（缓存）

### 数据库设计原则
1. 所有表使用`id`作为主键，自增
2. 外键关联通过代码层面维护，数据库不强制外键约束
3. 统一使用`created_at`和`updated_at`记录时间戳
4. 枚举值使用字符串存储（如warning_level: red/yellow/blue）

---

## 六、注意事项

1. **数据权限**：学生只能查看自己的数据，通过userId/studentId进行过滤
2. **隐私保护**：student_profile表的privacy_level字段控制数据可见范围
3. **成绩计算**：GPA由后端根据score_total和grade_point计算
4. **预警触发**：预警由系统根据warning_rules表的规则自动生成
5. **申诉流程**：学生提交申诉 → 教师审核 → 更新预警状态
