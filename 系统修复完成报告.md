**状态**: 已修复

修复了 `init_database.sql` 中的用户ID映射错误：

```sql
修改前:
- teacher_profile.user_id = 2 (❌错误，id 2是student)
- counselor_profile.user_id = 3 (❌错误，id 3是teacher)
- admin_profile.user_id = 4 (❌错误，id 4是counselor)

修改后:
- teacher_profile.user_id = 3 (✅正确，user_id=3是teacher001)
- counselor_profile.user_id = 4 (✅正确，user_id=4是counselor001)
- admin_profile.user_id = 5 (✅正确，user_id=5是admin001)
```

## 🚀 执行初始化数据

### 方案1: 使用MySQL命令行（推荐）

```bash
# 进入数据库目录
cd D:\ABS\学业预警系统后端\src\main\resources

# 执行主库初始化脚本（已修复）
mysql -u root -p123456 < init_database.sql

# 执行业务数据初始化脚本（新建）
mysql -u root -p123456 academic_warning_system < init_business_data.sql
```

### 方案2: 使用MySQL Workbench

1. 打开MySQL Workbench
2. 连接到本地MySQL服务器
3. File → Open SQL Script → 选择 `init_database.sql`
4. 执行（Ctrl+Shift+Enter 或 点击执行按钮）
5. 再执行 `init_business_data.sql`

### 方案3: 使用DataGrip或其他IDE

1. 打开IDE中的数据库连接
2. 右键点击连接 → Execute SQL Script
3. 选择 `init_database.sql`，执行
4. 再执行 `init_business_data.sql`
