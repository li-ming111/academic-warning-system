# 完整后端实现计划

## 需要实现的内容

### 教师端 (TeacherController + TeacherService)
1. POST /teachers/register - 教师注册
2. GET /teachers/dashboard/{userId} - 教学数据概览
3. POST /teachers/scores - 批量录入成绩
4. PUT /teachers/scores/{scoreId} - 修改成绩（记录score_logs）
5. GET /teachers/scores - 查询学生成绩列表
6. POST /teachers/scores/export - 导出成绩
7. GET /teachers/feedbacks/{teacherId} - 查询反馈
8. PUT /teachers/feedbacks/{feedbackId} - 回复反馈
9. GET /teachers/enrollments - 查询选修课
10. POST /teachers/communications - 保存沟通记录
11. GET /teachers/warnings/{teacherId} - 查询预警
12. PUT /teachers/warnings/{warningId}/process - 处理预警

### 辅导员端 (CounselorController + CounselorService)
1. POST /counselors/register - 辅导员注册
2. GET /counselors/dashboard/{userId} - 学生概览
3. GET /counselors/students - 学生列表
4. GET /counselors/students/{studentId} - 学生详情
5. POST /counselors/students/notify - 批量通知
6. GET /counselors/warnings - 预警列表
7. PUT /counselors/warnings/{warningId}/process - 处理预警
8. GET /counselors/enrollments - 学生选修课

### 管理员端 (AdminController + AdminService)
1. POST /admin/register - 管理员注册
2. GET /admin/dashboard - 综合数据概览
3. POST /admin/colleges - 创建学院
4. GET /admin/colleges - 查询学院
5. PUT /admin/colleges/{collegeId} - 修改学院
6. DELETE /admin/colleges/{collegeId} - 删除学院
7. POST /admin/majors - 创建专业
8. GET /admin/majors - 查询专业
9. PUT /admin/majors/{majorId} - 修改专业
10. DELETE /admin/majors/{majorId} - 删除专业
11. GET /admin/users - 用户列表
12. POST /admin/users - 创建用户
13. PUT /admin/users/{userId} - 修改用户
14. POST /admin/users/{userId}/disable - 禁用用户
15. POST /admin/users/{userId}/enable - 启用用户
16. GET /admin/rules - 查询规则
17. POST /admin/rules - 创建规则
18. PUT /admin/rules/{ruleId} - 修改规则
19. DELETE /admin/rules/{ruleId} - 删除规则
20. GET /admin/statistics - 数据统计

### 学生端 (补齐)
1. POST /students/register - 学生注册（含学号解析）
2. GET /students/dashboard/{userId} - 学业看板
3. GET /students/scores/{userId} - 成绩列表
4. GET /students/warnings/{userId} - 预警列表
5. GET /students/assistance/{userId} - 帮扶计划
6. POST /students/appeals - 提交申诉
7. GET /students/appeals/{userId} - 查询申诉
8. POST /students/messages - 发送消息
9. GET /students/messages/{userId} - 查询消息
10. POST /students/settings/{userId}/change-password - 修改密码
11. GET /students/statistics/{userId} - 统计分析

总计：约 50+ 个 API 端点，需要对应的 Service 和数据库操作

## 实现策略

1. **快速模板生成** - 为每个端点生成标准代码模板
2. **批量应用** - 使用代码生成工具批量创建
3. **增量调试** - 逐个用户端测试验证

预计完成时间：8-12 小时的代码生成和测试
