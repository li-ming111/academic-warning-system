# 完整后端代码生成-高效实现方案

## 核心策略：模板化生成所有后端代码

鉴于工作量巨大（50+个API端点跨4个用户端），采用以下高效策略：

### 1. 已完成的模板（教师端）
- TeacherService 接口：12个方法
- TeacherServiceImpl：完整实现所有方法
- TeacherController：所有API端点（register/dashboard/scores/feedbacks/enrollments/communications/warnings）

### 2. 待完成的（3个端）
- CounselorService/CounselorServiceImpl/CounselorController
- AdminService/AdminServiceImpl/AdminController  
- StudentService（补齐）/StudentServiceImpl（补齐）/StudentController（补齐）

### 3. 高效实现步骤

**步骤A：复制模板**
- 使用 TeacherService 作为模板
- 按照用户端需求调整方法名和参数
- 复制到 CounselorService/AdminService/StudentService

**步骤B：批量修改**
- 修改类名（Teacher -> Counselor/Admin/Student）
- 修改路径（/teachers -> /counselors/admin/students）
- 修改角色（role=2 -> 4/3/1）

**步骤C：前端API更新**
- 补齐 src/api/index.js 中所有API调用
- 连接所有前端页面到后端

**步骤D：测试验证**
- 编译检查
- 端到端测试
