## 1. 学生端模块

### 1.1 智能注册核心表
- ① 输入学号：注册页输入10位学号（如 `2023123456`）。截取第5-6位（例如 `34`），查询 `majors.code = '34'`。
- ② 自动填充专业：若存在匹配，展示“计算机科学与技术”+“信息工程学院”，并自动填充 `student_profile.major_id` 与 `student_profile.college_id`。
- ③ 填写手机号/邮箱：手机号（11位纯数字）与邮箱（含 `@` 和域名）校验通过后写入 `users.phone` 与 `users.email`。
- ④ 提交完成：创建 `users` 记录（`role = 1`），并建立 `users.id ↔ student_profile.user_id` 外键关联。

数据联动要点：
- `majors.code` 字段匹配学号第5-6位 → 反查 `majors.id` 与 `majors.college_id` → 写入 `student_profile.major_id` 与 `student_profile.college_id`。

### 1.2 学业全景看板
- 登录后点击“学业看板”。聚合当期 `scores` 成绩与 `academic_warnings` 预警统计。
- 课程数与 GPA：显示“本学期课程数：8门”“GPA：3.2”，计算公式：`SUM(grade_point * credits) / SUM(credits)`（联动 `scores.grade_point` 与 `courses.credits`）。
- 预警状态：显示“黄色预警 ×2”与红点提醒，查询 `academic_warnings.status = 'confirmed'` 记录数。
- 关键关联：`scores.student_id ↔ student_profile.user_id`。

### 1.3 成绩深度分析
- 成绩趋势：学期筛选（如 `semester = '2023-2024秋'`）筛选 `scores` 数据。
- 趋势高亮：如《高等数学》“75→68”，将 `scores.score_total < 70` 的记录标红并触发预警逻辑。
- 申诉通道：点击“申诉”，填写理由（如“缓考未同步”）并上传 PDF。写入 `score_appeals`，状态设为 `pending`，`score_appeals.warning_id ↔ academic_warnings.id`。

### 1.4 智能预警提醒
- 通知：显示“【红色预警】《线性代数》62分”，查询 `academic_warnings.warning_level = 'red'`。
- 处理建议：详情展示 `academic_warnings.description`（如“本周加课辅导”）。
- 申诉预警：提交“成绩录入错误”与截图，更新 `academic_warnings.appeal_status = 'pending'`。

### 1.5 帮扶计划追踪
- 查看帮扶计划：列表查询 `assistance_plans.student_id = 当前用户ID`（`↔ student_profile.user_id`）。
- 更新进度：拖动进度条到 70%，填写“已完成专项练习”，更新 `assistance_plans.progress_percentage = 70`。
- 通知教师：提交后写入 `notifications`，`notifications.user_id = teacher_profile.user_id`。

### 1.6 个人信息管家
- 修改密码：输入原密码与新密码（≥8位），`SHA-256 + 盐值` 加密后写入 `users.password`。
- 隐私权限：设置“仅教师可见成绩”，更新 `student_profile.privacy_level = 2`。
- 安全日志：显示最近 5 次登录 IP，读取 `security_log.ip_address` 与 `security_log.login_time`（`security_log.user_id ↔ student_profile.user_id`）。

## 2. 教师端模块

### 2.1 教师注册
- 输入工号（如 `202312`），作为 `users.username` 唯一标识。
- 选择学院：查询 `colleges.name = '人工智能学院'` → 获取 `college_id = 201`，写入 `teacher_profile.college_id = 201`。
- 设置密码：≥8位，`SHA-256` 加密写入 `users.password`。
- 联系方式：校验并写入 `users.phone`、`users.email`。
- 提交注册：创建 `users`（`role = 2`）并关联 `teacher_profile`（示例 `users.id = 101 ↔ teacher_profile.user_id = 101`）。

### 2.2 教学数据概览
- 关键数据：聚合 `student_profile.college_id = 201` 显示“学生总数 120 人；预警：红色 ×3，黄色 ×5”。
- 预警列表：查询 `academic_warnings.warning_level = 'red'` 且 `status = 'pending'`。
- 待办事项：从 `notifications` 读取 `user_id = 当前教师ID` 的待办。
- 申诉预警：读取 `score_appeals.status = 'under_appeal'`（`warning_id ↔ academic_warnings.id`）。

### 2.3 学生成绩管理
- 批量录入：上传 Excel（含学生ID/平时分/期末分），按 `courses.score_template` 计算总分：`scores.score_total = (平时分 × 0.3 + 期末分 × 0.7)`。
- 考核模板：选择“考试课（平时 30% + 期末 70%）”，更新 `courses.score_template = 'exam'`。
- 修改成绩：录入新分数与原因，记录 `score_logs`（`modified_by = 教师ID`，`reason`）。
- 导出成绩：导出含学号/成绩/课程的 Excel（映射 `scores.student_id`、`scores.score_total`、`scores.course_id`）。

### 2.4 学业预警管理（重点升级）
- 查看预警：如学生B《高数》52分（红色预警），查询 `academic_warnings.student_id = 2001`。
- 处理预警：填写帮扶措施“加课辅导 + 每周答疑”，更新 `academic_warnings.status = 'processed'`。
- 记录沟通（新增）：输入沟通内容（如“家庭原因缺课，需心理疏导”），写入 `communication_logs`。
- 跟踪帮扶：更新 `assistance_plans.progress_percentage = 70`。
- 申诉处理：查看 `score_appeals.reason = '缓考申请未同步'`（`↔ academic_warnings.id`）。

### 2.5 学生表现分析
- 平均分：选择课程A → 显示 `AVG(scores.score_total)`（示例 78.5），`scores.course_id = 1001`。
- 成绩分布：柱状图查看分段人数，统计 `scores.score_total` 分段（`GROUP BY score_total DIV 10`）。
- 学生详情：查询该生所有课程成绩（`scores.student_id = 2001`）。
- 风险识别：如《线代》62分自动标记，触发 `academic_warnings` 创建（`student_id = 2001`）。

### 2.6 学生反馈管理
- 查看反馈：查询 `feedbacks.teacher_id = 101`，显示“课程内容反馈 ×5”。
- 分类：按 `feedbacks.category = '教学方法'` 过滤。
- 回复：写入 `feedbacks.reply_content`（如“已优化PPT，增加案例讲解”）。
- 满意度：聚合 `feedbacks.rating`（1-5分），计算 `AVG(feedbacks.rating)`。

### 2.7 学生选修课信息查看
- 查看选修课：查询 `enrollments` 中 `courses.type = '选修'`，显示“已选《Python》（75分）”。
- 学分核查：`SUM(courses.credit)` 显示“已修 6 学分/需 8 学分”。
- 推荐选课：查询 `courses.recommended_for = student_profile.research_areas`，示例显示《机器学习基础》。
- 沟通留痕（新增）：输入“学生对 NLP 感兴趣，推荐《自然语言处理》”，写入 `communication_logs`。

## 3. 辅导员端模块

### 3.1 辅导员注册
- 输入 10 位工号（如 `2023123456`），写入 `users.username`。
- 选择学院：查询 `colleges.name = '信息工程学院'`，获取 `colleges.id = 101`，关联 `counselor_profile.college_id = 101`。
- 设置密码：≥8位，`SHA-256 + 盐值` 加密写入 `users.password`。
- 联系方式：校验并写入 `users.phone` 与 `users.email`。
- 提交完成：创建 `users`（`role = 4`）并关联 `counselor_profile`（`users.id ↔ counselor_profile.user_id`）。

### 3.2 学生情况概览
- 首页数据：聚合 `student_profile.college_id = 101` 的学生数据（`↔ counselor_profile.college_id`）。
- 预警趋势：按月视图显示“红色预警上升趋势”，查询 `academic_warnings.college_id = 101`。
- 成绩分布：选择“班级A”，显示“平均分 75，优秀率 20%”（`scores.class_id ↔ classes.id`）。
- 待办事项：从 `notifications` 读取 `user_id = 当前辅导员ID` 的待办。

### 3.3 学生管理
- 学生列表：筛选“信息工程学院+2023级”，显示“班级 A 至 G”，查询 `student_profile.college_id = 101 AND class_id IN (1001-1007)`。
- 搜索学生：模糊搜索 `student_profile.name LIKE '%张三%'`。
- 学生详情：查询 `student_profile.id = 2001` 的完整记录（`↔ student.id`）。
- 批量通知：选择“班级B”，写入 `notifications`（`user_id = 2001-2010`）。
- 导出学生信息：导出 `student_profile` 的 `student_id, name, phone` 等列。

### 3.4 学生预警管理
- 预警列表：查询 `academic_warnings.college_id = 101 AND status = 'pending'`。
- 处理预警：填写处理意见“加课辅导 + 心理疏导”，更新 `academic_warnings.status = 'processed'`，并写入 `assistance_plans`（`warning_id ↔ academic_warnings.id`）。
- 分配协作：写入 `notifications`（`user_id = 教师X ID`）。
- 跟踪进度：更新 `assistance_plans.progress_percentage = 70` 并实时展示。

### 3.5 学生选修课信息查看
- 班级修读概览：选择“班级A”，显示“选修课完成率 85%”，查询 `enrollments.class_id = 1001 AND courses.type = '选修'`。
- 未达标预警：筛选 `SUM(courses.credit) < 8` 的记录，提示“学生E学分不足”。
- 修读详情：查询 `enrollments.student_id = 2002`，显示“已修《Python》（70分）”。

## 4. 管理员端模块

### 4.1 管理员注册
- 输入 10 位工号（如 `2023123456`），写入 `users.username`。
- 选择部门：查询 `admin_profile.department = '教务处'` → 获取 `department.id = 101`，写入 `admin_profile.department`。
- 设置密码：≥8位，`SHA-256 + 盐值` 加密写入 `users.password`。
- 联系方式：校验并写入 `users.phone` 与 `users.email`。
- 提交完成：创建 `users`（`role = 3`）并关联 `admin_profile`（`users.id ↔ admin_profile.user_id`）。

### 4.2 综合数据概览
- 首页：聚合 `student_profile`、`academic_warnings`、`assistance_plans` 等表数据（如 `student_profile ↔ academic_warnings.student_id`）。
- 今日概览：显示“今日新增预警 ×5”“处理预警 ×3”，过滤 `academic_warnings.created_at = 今天`。
- 最新预警：显示 `academic_warnings.status = 'pending'` 的记录。
- 系统动态：查询 `audit_logs.action_type = 'update'`，`audit_logs.user_id ↔ users.id`。

### 4.3 学院管理页（系统基石）
- 添加学院：创建 `colleges`（如 `name = '人工智能学院'`），`colleges.id` 自动生成（如 `201`）。
- 修改学院：更新 `colleges.name = '信息工程学院'`（`id = 101`）。
- 删除学院：先校验并清空 `student_profile.college_id = 201` 与 `teacher_profile.college_id = 201` 引用，再删除，避免数据孤岛。
- 查看学院列表：查询 `colleges`，按 `id` 排序展示。

### 4.4 专业编码管理页（学号自动解析引擎）
- 配置专业映射：写入 `majors`（示例 `code = '02', name = '软件工程', college_id = 301`）。
- 学号自动解析：学号 `2023020627` → 截取第 5-6 位 `'02'` → 查询 `majors.code = '02'` → 自动显示专业并获取 `college_id = 301`。
- 修改专业映射：更新 `majors.name = '计算机科学'`（`code = '02'`）。
- 删除专业编码：先清空 `student_profile.major_id = 03` 的关联，再删除对应 `majors`。

### 4.5 用户信息监管页（异常修正中枢）
- 按学院筛选：`student_profile.college_id = 101` 与 `teacher_profile.college_id = 101`。
- 按角色筛选：`users.role = 4`（辅导员）。
- 修正用户学院：更新 `student_profile.college_id = 201`（`user_id = 2005`）。
- 批量导出用户：导出 `users` 与 `teacher_profile`（关联 `users.id ↔ teacher_profile.user_id`）。

### 4.6 用户管理
- 查看用户列表：过滤 `users.role = 2`，显示“教师A（工号201）”。
- 搜索用户：模糊 `users.name LIKE '%张三%'`。
- 添加新用户：写入 `users`（`role = 4`）并在 `admin_profile` 建立关联（`users.id ↔ admin_profile.user_id`）。
- 编辑用户信息：更新 `users.email`（如 `users.id = 201`）。
- 批量禁用账户：更新 `users.status = 0`。

### 4.7 预警规则管理（核心闭环）
- 查看规则：读取 `warning_rules`（`condition` 存储规则逻辑）。
- 添加规则：示例“挂科科目数量分级预警”
  - `SUM(failed_courses) <= 3 → level = 'low'`
  - `3 < SUM(failed_courses) <= 5 → level = 'medium'`
  - `SUM(failed_courses) > 5 → level = 'high'`
  - 写入 `warning_rules.condition` 与 `warning_rules.level`
- 测试规则：选择学生Z，模拟挂科 4 科 → 触发 `academic_warnings` 写入 `warning_level = 'medium'`（`rule_id ↔ warning_rules.id`）。
- 禁用旧规则：更新 `warning_rules.status = 0`。

### 4.8 数据统计
- 学生统计：聚合 `student_profile.major` 分组，显示“软件工程专业学生 ×200”。
- 预警统计：聚合 `academic_warnings.level` 分组，显示“红色 ×15，黄色 ×30”。
- 帮扶报表：按“2025年春季学期”过滤 `assistance_plans.completed_at` 并导出 Excel。
- 系统使用：聚合 `audit_logs.action_type = 'login'` 次数（如“登录次数 ×5000”）。

### 4.9 学生选修课数据中心
- 配置毕业要求：写入 `elective_requirements`（如 `grade = 2021` 要求修满 8 学分）。
- 查询学生记录：输入学号 `2023020627` → 查询 `enrollments`（`↔ student_profile.id`），显示“已修《Python》（70分）”。
- 生成全校统计：聚合 `enrollments`，过滤 `course_type = '选修'`，显示“2023级选修课完成率 85%”。
- 导出未达标名单：筛选 `credit_sum < 8` 并导出 Excel。

## 5. 预警级别与规则联动

| 挂科科目数 | 预警级别 | 系统提示语（academic_warnings.description） |
| --- | --- | --- |
| ≤ 3 科 | 低级 | 挂科 {数量} 科，建议补考 + 基础辅导 |
| 3–5 科 | 中级 | 挂科 {数量} 科，需制定帮扶计划 |
| > 5 科 | 高级 | 挂科 {数量} 科，已触发高危预警，请辅导员介入 |

规则与数据表联动（闭环）：
- `warning_rules.condition`：存储规则逻辑（如 `SUM(failed_courses) <= 3`）。
- `warning_rules.level`：预警级别标识（`low/medium/high`），直接决定显示样式。
- `academic_warnings.warning_level`：实际预警级别（系统自动生成）。
- `academic_warnings.description`：预警提示文本（系统自动填充）。

## 7. 异常场景：学号解析失败
- 场景：学号 `2023020627` → 未找到 `majors.code = '02'`。
- 系统行为：
  1. 提示“专业编码未配置，请联系管理员”。
  2. 自动记录 `audit_logs`（操作人 = 学生，时间 = 当前，内容 = “学号解析失败”）。
  3. 生成待处理工单（状态 = 待配置，优先级 = 高）。

## 9. 数据表清单

### 9.1 学生端数据表
- `majors`：专业编码匹配（学号第 5-6 位）→ 智能注册。
- `users`：账号管理（密码、手机号、邮箱）→ 个人信息管家。
- `student_profile`：学生档案（学院、专业、隐私）→ 智能注册、看板、信息管家。
- `scores`：成绩数据（学期、课程、分数、学分）→ 深度分析、全景看板。
- `academic_warnings`：预警数据（等级、描述、申诉状态）→ 预警提醒、深度分析。
- `score_appeals`：申诉记录（理由、证明）→ 深度分析（申诉通道）。
- `assistance_plans`：帮扶计划（进度、状态）→ 帮扶计划追踪。
- `notifications`：通知记录（预警、进度更新）→ 预警提醒、帮扶追踪。
- `security_log`：安全日志（登录IP、时间）→ 个人信息管家。

### 9.2 教师端数据表
- `users`：教师身份（工号、密码、联系方式）→ 注册、认证、联系方式。
- `teacher_profile`：职业信息（学院、职称、研究方向）→ 注册学院归属、学院维度统计。
- `courses`：课程信息（编号、名称、考核类型、学分）→ 成绩模板与课程管理。
- `scores`：学生成绩数据（课程ID、学生ID、分数、学分）→ 录入/修改/导出、表现分析。
- `score_logs`：成绩修改日志（修改人、时间、原因）→ 留痕合规。
- `academic_warnings`：学业预警（学生ID、等级、状态、描述）→ 创建/处理/追踪。
- `assistance_plans`：帮扶计划（目标、措施、进度、关联预警ID）→ 制定与追踪。
- `communication_logs`（新增）：沟通记录（教师ID、学生ID、内容、时间）→ 预警处理留痕 + 选修建议归档。
- `feedbacks`：学生反馈（内容、分类、回复、关联教师ID）→ 收集/分类/回复。

## 10. 绑定与查询过滤关键逻辑

关键绑定：
- 教师-班级绑定：`classes.teacher_id = teacher_profile.user_id`。
- 学生-班级绑定：`student_profile.class_id = classes.id`。
- 辅导员-班级绑定：`classes.counselor_id = counselor_profile.user_id`。

过滤逻辑（示例）：
- 教师视角：`WHERE student_profile.class_id IN (SELECT id FROM classes WHERE teacher_id = 教师ID)`。
- 辅导员视角：`WHERE student_profile.class_id IN (SELECT id FROM classes WHERE counselor_id = 辅导员ID)`。

## 安全与合规要点
- 密码加密：统一采用 `SHA-256 + 盐值`，禁止明文存储。
- 外键约束：删除学院/专业编码前，先清理关联引用，避免孤儿数据。
- 修改留痕：成绩修改须写入 `score_logs` 并保留原因，满足教育规范。
- 隐私控制：`student_profile.privacy_level` 控制成绩可见范围（如仅教师可见）。

## 闭环与状态机（增强）

### 闭环流程概览
- 触发：成绩录入/修改、规则变更、毕业要求核查。
- 检测：规则引擎计算失败科目与等级，生成/更新 `academic_warnings`。
- 通知：写入 `notifications` 并通过站内、短信、邮件分发，带送达确认。
- 干预：教师/辅导员制定 `assistance_plans`，创建沟通记录 `communication_logs`。
- 跟踪：进度更新与逾期提醒，面向学生与干预方可视化。
- 复盘：处理完成后关闭预警，生成审计与报表，规则优化迭代。

### 状态机定义
- `academic_warnings.status`：`pending` → `confirmed` → `processed` → `closed`
- `academic_warnings.appeal_status`：`none` → `pending` → `under_appeal` → `resolved | rejected`
- `assistance_plans.status`：`initiated` → `in_progress` → `completed` → `archived`
- `notifications.status`：`unread` → `read`；增加送达日志与失败重试计数

状态迁移约束：
- 当 `scores` 更新导致风险解除时，允许直接将 `academic_warnings` 置为 `closed`；若存在 `appeal_status = under_appeal`，需先结案。
- 当 `assistance_plans` 完成且对应预警为 `processed`，可自动触发关闭与复盘报告生成。

## 触发器与定时任务

- 成绩事件触发：
  - 新增/修改 `scores` → 重新计算 GPA 与规则引擎 → 更新/创建 `academic_warnings`。

- 规则事件触发：

- 定时任务：
  - 每日 02:00 重算逾期帮扶与预警统计，生成日报。
  - 每周一生成班级/学院报表并归档到审计存储。

- 事务边界：成绩更新 → 规则计算 → 预警写入 → 帮扶初始化采用事务或补偿（saga）。

## 设计与实现概述（Spring Boot + Vue3）

- 核心技术：`Spring Boot` + `Vue 3` 前后端分离；`MySQL`（学生学业数据/预警记录），`Redis`（缓存学业指标阈值），`ECharts`（数据可视化），`Java Mail`（预警通知与帮扶提醒），`Apache POI`（帮扶计划导出）。
- 后端设计：
  - 服务框架：Spring Boot 3 提供统一依赖与自动配置，模块化划分用户/成绩/预警/帮扶/通知。
  - 数据访问：MyBatis-Plus 管理实体与 CRUD，分页与条件查询支撑看板与列表性能。
  - 数据库：MySQL 存储结构化数据，外键与索引按“索引与查询优化建议”执行。
  - 安全：Spring Security 统一认证授权，与角色矩阵绑定资源访问；支持基于 JWT 的会话。
  - 缓存：Redis 缓存规则阈值与热点统计；预警列表缓存仅 `status=pending`，提升响应。
  - 通知：Java Mail 发送邮件通知；与站内消息统一走 outbox 异步分发与送达日志。
  - 导出：Apache POI 生成帮扶计划与成绩报表（支持分页流式写出）。
- 前端实现：
  - 框架：Vue 3 + Vite，Element Plus 构建交互界面（注册/看板/分析/预警/帮扶）。
  - 通信：Axios 封装 API，请求/响应拦截对接鉴权与错误处理。
  - 可视化：ECharts 绘制“挂科率趋势”“成绩分布”“预警趋势”等图表。
- 关键联动：
  - 规则阈值：从 Redis 读取预警阈值（如挂科科目分级），缺失回退至 MySQL；规则更新触发阈值缓存刷新。
  - 预警通知：规则触发 → 写入 `academic_warnings` → outbox 分发站内+邮件 → 送达确认与失败重试。
  - 导出闭环：帮扶进度完成 → 生成 Excel 报表（POI） → 归档审计与下载权限控制。
